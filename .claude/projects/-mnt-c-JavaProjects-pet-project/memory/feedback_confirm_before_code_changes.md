name: Confirm before code changes
description: Never change code without explicit user confirmation first
type: feedback

Never change any code without the user's explicit confirmation first. Always explain the proposed change and wait for approval before editing.

**Why:** User explicitly stated this preference after code was changed without asking.

**How to apply:** When a code change is needed, describe the change clearly, then ask for confirmation before using Edit/Write tools on any source files.
